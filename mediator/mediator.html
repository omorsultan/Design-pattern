Mediator Pattern (বাংলায়)

একটি Behavioral Design Pattern

যখন অনেক object একে অপরের সাথে directly communicate করে এবং code খুব জটিল হয়ে যায়, তখন একটি Mediator object ব্যবহার করা হয়।

সব object নিজেদের মধ্যে সরাসরি কথা না বলে Mediator-এর মাধ্যমে যোগাযোগ করে。

Real Life Example

Air Traffic Control (ATC)

একটি Airport-এ অনেক Plane আছে।

❌ Plane গুলো যদি একে অপরের সাথে directly communicate করে:

Plane A <--> Plane B
Plane A <--> Plane C
Plane B <--> Plane C

যোগাযোগ ব্যবস্থা খুব জটিল হয়ে যাবে।

✔ সব Plane ATC Tower-এর সাথে communicate করে:

Plane A --> ATC
Plane B --> ATC
Plane C --> ATC

ATC = Mediator

Messy Code (Without Mediator)

ধরি একটি Chat System আছে।

class User {
    private String name;

    public User(String name) {
        this.name = name;
    }

    public void sendMessage(User user, String message) {
        System.out.println(name + " sends to " + user.name + ": " + message);
    }
}

public class Main {
    public static void main(String[] args) {
        User omor = new User("Omor");
        User rahim = new User("Rahim");
        User karim = new User("Karim");

        omor.sendMessage(rahim, "Hello");
        omor.sendMessage(karim, "Hi");
        rahim.sendMessage(karim, "How are you?");
    }
}

Problems

  • User একে অপরকে directly জানে
  • Coupling বেশি
  • User সংখ্যা বাড়লে communication complex হবে
  • Group chat implement করা কঠিন

Mediator Pattern Solution

Mediator Interface

interface ChatMediator {
    void sendMessage(String message, User sender);
    void addUser(User user);
}

Concrete Mediator

import java.util.ArrayList;
import java.util.List;

class ChatRoom implements ChatMediator {
    private List<User> users = new ArrayList<>();

    @Override
    public void addUser(User user) {
        users.add(user);
    }

    @Override
    public void sendMessage(String message, User sender) {
        for (User user : users) {
            if (user != sender) {
                user.receive(message);
            }
        }
    }
}

User Class

class User {
    private String name;
    private ChatMediator mediator;

    public User(String name, ChatMediator mediator) {
        this.name = name;
        this.mediator = mediator;
    }

    public String getName() {
        return name;
    }

    public void send(String message) {
        System.out.println(name + " sends: " + message);
        mediator.sendMessage(message, this);
    }

    public void receive(String message) {
        System.out.println(name + " received: " + message);
    }
}

Main Class

public class Main {
    public static void main(String[] args) {
        ChatRoom room = new ChatRoom();

        User omor = new User("Omor", room);
        User rahim = new User("Rahim", room);
        User karim = new User("Karim", room);

        room.addUser(omor);
        room.addUser(rahim);
        room.addUser(karim);

        omor.send("Hello Everyone");
    }
}

Output:

Omor sends: Hello Everyone

Rahim received: Hello Everyone

Karim received: Hello Everyone

UML Diagram & Relations

+------------------+ +----------------+ | ChatMediator |<-----------------| User | +------------------+ +----------------+ | + sendMessage() | | - mediator | | + addUser() | +----------------+ +------------------+ | + send() | ^ | + receive() | | (Implementation) +----------------+ | ^ +------------------+ | | ChatRoom | | +------------------+ | | - users: List |--------------------------+ +------------------+ (Aggregation: o--) | + sendMessage() | | + addUser() | +------------------+

Relation Types

  • 1. Implementation: ChatRoom implements ChatMediator
  • 2. Association: User --> ChatMediator (User mediator ব্যবহার করে)
  • 3. Aggregation: ChatRoom o--> User (ChatRoom অনেক User maintain করে)

কখন ব্যবহার করবেন?

  • Chat Application
  • Air Traffic Control System
  • Smart Home Devices
  • GUI Components Communication
  • Online Meeting System

Mediator Pattern-এর মূল ধারণা

আগে (Before)

User A <--> User B
User A <--> User C
User B <--> User C

সবাই সবাইকে চিনে।

পরে (After)

User A --> ChatRoom
User B --> ChatRoom
User C --> ChatRoom

কেউ কাউকে directly চেনে না। সব communication Mediator-এর মাধ্যমে হয়।
